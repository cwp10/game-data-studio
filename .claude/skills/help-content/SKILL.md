## 도움말 콘텐츠 원칙

대상 독자: **게임 데이터 기획자** (비개발자). 수식이나 코드보다 "이 버튼을 누르면 무슨 일이 생기는가"를 명확히 설명한다.

각 화면 설명은 아래 구조를 따른다:
```
제목: {화면 이름}
부제: {한 줄 목적 요약}
기능 목록: [{ icon?, title, desc }]  // 3~5개
팁: {선택적, 핵심 워크플로우 한 줄}
```

## 8개 화면 핵심 설명 (콘텐츠 초안)

### 프로젝트 (home)
- **목적**: 게임 프로젝트를 생성하고 관리하는 시작점
- **기능**:
  - 새 프로젝트 생성 — RPG 장르 선택 후 자동으로 테이블 구조가 만들어짐
  - 프로젝트 카드에서 테이블 수·행 수 한눈에 확인
  - 프로젝트 삭제 (카드 우측 버튼)
- **팁**: 장르를 선택하면 수집형 RPG·방치형 RPG 등에 맞는 기본 테이블이 자동 생성됩니다

### 스키마 (schema)
- **목적**: 테이블 구조(컬럼)를 설계하는 화면
- **기능**:
  - 테이블 추가·삭제
  - 컬럼 추가·수정 (이름, 타입: 문자/숫자/불리언, 제약 조건)
  - 테이블 간 관계 설정 (FK 연결)
  - 컬럼 제약 — 최솟값/최댓값/필수/고유 설정
- **팁**: 컬럼 타입을 `enum`으로 지정하면 타입 탭에서 선택지를 정의할 수 있습니다

### 데이터 (editor)
- **목적**: 테이블에 실제 수치 데이터를 입력·편집하는 스프레드시트
- **기능**:
  - 셀 직접 편집 (한글 입력 지원)
  - Ctrl+Z / Ctrl+Shift+Z 실행 취소·재실행
  - 복수 셀 선택 후 Ctrl+C/V로 복사·붙여넣기
  - CSV 가져오기·내보내기 / JSON 내보내기
  - 이상값 하이라이팅 (빨간/노란 셀) — 밸런스가 깨진 셀을 자동 감지
  - 스냅샷(버전 저장) 및 버전 간 차이 비교
  - 행 메모 — StickyNote 아이콘으로 수치 근거 기록
- **팁**: 곡선 모달(📈 버튼)로 레벨별 수치를 자동으로 채울 수 있습니다

### 타입 (types)
- **목적**: 컬럼에서 사용하는 선택지(enum) 목록을 관리
- **기능**:
  - enum 타입 생성·수정·삭제
  - 타입별 허용값 목록 정의 (예: grade → SSR, SR, R, N)
  - 컬럼에 타입을 지정하면 데이터 편집 시 드롭다운으로 선택
- **팁**: 등급·직업·속성 등 반복 사용되는 값은 타입으로 만들어두면 일관성을 유지할 수 있습니다

### 밸런싱 (balance)
- **목적**: 테이블 수치의 이상값을 감지하고 밸런스를 분석
- **기능**:
  - 컬럼별 평균·분포·이상값 자동 감지 (MAD 기반)
  - Danger(빨강)/Warn(노랑) 이상값 목록
  - 능력치 레이더 차트 — 유닛 간 능력치 비교
  - Pearson 상관계수 — 컬럼 간 상관관계 분석
  - 승률 매트릭스 — 유닛 간 PvP 승률 예측
  - AI 밸런스 리포트 — Claude가 수치를 분석하고 개선안 제시
- **팁**: 이상값 셀을 클릭하면 해당 데이터 행으로 바로 이동합니다

### 시뮬레이션 (simulation)
- **목적**: 수치를 시뮬레이션해 게임 플레이 감을 예측
- **기능**:
  - **저장된 시뮬** — 실제 DB 데이터로 수식 자동 도출
  - **스탯 계산기** — 레벨별 최종 스탯 비교
  - **전투** — 몬테카를로 기법으로 승률·HP 추이 시뮬
  - **가챠** — 소프트 천장 포함 가챠 확률 분포
  - **DPS** — 빌드별 초당 데미지 비교
  - **난이도** — 스테이지별 승률·플레이타임 예측
  - **페이싱** — N일차 레벨/스테이지/재화 타임라인
- **팁**: 페이싱 시뮬로 콘텐츠 소진 속도와 페이 게이트 위치를 미리 확인할 수 있습니다

### 경제 (economy)
- **목적**: 재화 흐름(수입·지출)을 시뮬레이션
- **기능**:
  - 재화 항목별 일일 수입·지출 입력
  - 성장률·인플레이션 설정
  - 잔액 추이 LineChart — 실질/명목 잔액 비교
- **팁**: 인플레이션을 설정하면 재화 가치 하락 곡선을 시각화할 수 있습니다

### 메모리 (memory)
- **목적**: Claude AI와 대화하며 DB를 자연어로 조작하고, 프로젝트 맥락을 저장
- **기능**:
  - AI 채팅 — "영웅 테이블에 레벨 50 데이터 추가해줘" 같은 자연어 명령 실행
  - 프로젝트 메모 — 기획 의도·결정 사항을 자유형식 메모로 저장
  - 대화 기록이 다음 세션에도 유지됨
- **팁**: 테이블 구조가 복잡할 때 AI에게 "이 테이블 설명해줘"라고 물어볼 수 있습니다

## HelpModal 컴포넌트 설계

### 파일 위치
```
src/components/ui/HelpModal.tsx
```

### 권장 구조 (탭 + 콘텐츠 패널)
```tsx
interface HelpSection {
  id: Screen;          // 'home' | 'schema' | 'editor' | 'types' | 'balance' | 'simulation' | 'economy' | 'memory'
  label: string;       // 사이드바 표시 이름
  icon: React.ElementType;
  title: string;       // 도움말 제목
  subtitle: string;    // 한 줄 설명
  features: { title: string; desc: string }[];
  tip?: string;
}

// Modal 열기 방식: props로 초기 화면 지정 (컨텍스트 도움말)
interface HelpModalProps {
  open: boolean;
  onClose: () => void;
  initialScreen?: Screen;  // 현재 보고 있는 화면을 기본 선택
}
```

```tsx
"flex-1 p-6 overflow-y-auto"
```

### 도움말 버튼 연결 위치
각 화면 헤더 우측 또는 Sidebar 하단에 `?` 아이콘 버튼을 배치한다.

```tsx
// 방법 A: 각 화면 헤더
<button onClick={() => setHelpOpen(true)} className="...">
  <HelpCircle size={16} />
</button>

// 방법 B: Sidebar 하단 (전역 도움말)
// Sidebar.tsx 하단에 HelpCircle 아이콘 버튼 추가
// app/page.tsx에서 helpScreen state 관리
```

**권장: 방법 B** — Sidebar 하단 `?` 버튼 1개로 전역 도움말 열기. `initialScreen`에 현재 활성 화면을 전달해 컨텍스트 도움말처럼 동작.

## 콘텐츠 업데이트 패턴

화면 코드가 바뀌면 도움말도 함께 업데이트한다:

1. 변경된 화면 컴포넌트 읽기
2. 기존 `HelpModal.tsx`의 해당 섹션 찾기
3. `features` 배열과 `tip`만 교체 (구조 유지)
4. tsc 확인
