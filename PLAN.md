# 시뮬레이션 기획

## 목표
게임 엔진(Unity 등)을 실행하지 않고, 앱 안에서 수치를 미리 확인·검증한다.
C# 코드 출력은 필요 없음.

---

## 필요한 시뮬레이션 유형 (우선순위 순)

### 1. 스탯 계산기 (Stat Calculator)
"레벨 50, 강화 +7일 때 이 캐릭터의 스탯이 얼마인가"

**입력**
- 캐릭터 선택
- 레벨 (예: 1~50)
- 강화 단계 (예: +0~+10)

**출력**
- ATK, DEF, HP 등 최종 스탯 수치
- 조건별 비교 (레벨 1 vs 50, 강화 0 vs 7 등)

**핵심 공식**
```
최종 스탯 = 기본값 × 레벨 성장 보정 × 강화 보정
```

**미결정 사항**
- 데이터 구조 선택 필요:
  - A안: characters 테이블에 레벨별 행으로 저장 (level=1, atk=100 / level=50, atk=250)
  - B안: characters는 기본값만, 성장률은 별도 테이블 (level_curves)
  - 강화 테이블도 동일한 방식으로 결정 필요

---

### 2. 전투 시뮬레이션 (Battle Simulation)
"이 캐릭터로 이 스테이지를 클리어할 수 있나"

**입력**
- 캐릭터 스탯 (스탯 계산기 결과 연동 가능)
- 스테이지/적 스탯

**출력**
- 승/패, 소요 턴
- 결과 차트 (HP 추이 등)

---

### 3. 가챠 시뮬레이션 (Gacha Simulation)
"SSR 확률 3%면 평균 몇 번 뽑아야 하나"

**입력**
- 가챠 테이블 선택 (확률 데이터)
- 시뮬레이션 횟수 (예: 10,000번)

**출력**
- 평균 획득 횟수
- 확률 분포 그래프
- 천장(pity) 반영 여부

---

### 4. 성장 곡선 시각화 (Growth Curve Viewer)
"레벨업마다 스탯이 적절하게 오르고 있나"

**입력**
- 테이블 선택
- 시각화할 컬럼 선택 (ATK, DEF, HP 등)
- X축 기준 (레벨, 강화 단계 등)

**출력**
- 라인 차트

---

## UX 방향
- "새 시뮬레이션" → 유형 선택 → 데이터 연결 → 즉시 결과
- 결과는 앱 안에서 확인 (C# 코드 출력 없음)
- 저장해두고 나중에 다시 볼 수 있도록
